# U3 · STEP 01 ｜從訂單可視化走到 LINE Flex

> **這堂完成物**：把一筆訂單做成 LINE Flex Message，並真的推播到 LINE OA。

## 1. 先打開「訂單可視化」

啟動專案後，點上方「**訂單可視化**」。

你應該看到：

- 訂單狀態 3D 動畫
- 訂單流程板：接單、揀貨、覆核、出貨
- 每筆訂單的狀態、區域、金額、ETA

這一頁不是只看動畫。

老師要帶學生看懂：

```text
資料欄位 → React component → CSS 狀態樣式 → 互動畫面
```

也就是：資料不是放在表格裡等人讀，資料可以變成畫面。

## 2. 看今天要送出的訂單

打開：

```text
data-lab/orders.json
```

這份資料會拿去做「訂單資訊」Flex Message。

你要看懂這幾個欄位：

| 欄位 | 用途 |
|---|---|
| `order_id` | LINE 卡片上的訂單編號 |
| `customer` | 客戶名稱 |
| `channel` | 訂單來源 |
| `status` | 目前狀態 |
| `amount` | 訂單金額 |
| `items` | 商品明細 |

## 3. 切到「LINE 推播中心」

點「**LINE 推播中心**」。

推播範本選：

```text
訂單資訊
```

接下來照按鈕順序走：

```text
載入資料 → 檢查資料合約 → 生成 Flex 預覽 → 人工審核 → 推播 LINE Flex
```

先不要急著送。

這一段要讓學生先有「哇，訂單真的可以變成 LINE 卡片」的感覺。

## 4. 這堂不是 API 課

你只需要先懂三句話：

1. Flex Message 是 LINE 卡片的長相。
2. token 是 LINE OA 的鑰匙，不能放在前端。
3. 真正送出 LINE 的橋接程式已經寫好，今天重點是會使用、會檢查、會驗收。
